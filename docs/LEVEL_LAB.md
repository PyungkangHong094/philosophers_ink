- 위치: 순수 Dart CLI `tool/level_lab/` — 앱 코드(lib/)를 헤드리스로 재사용, 앱 빌드에 미포함.

## 0. 차곡 → 잉크 번역표

| 차곡 (이산 소코반) | 현자의 잉크 (연속 falling-sand) |

## 1. 파이프라인 5단계

### L1. 솔버 (탐색적 자동 플레이) — 최우선
- **스트로크 프리미티브**: 해 후보를 "선분 k개(각: 시작점·끝점·잉크종류)"로 파라미터화.
  k ≤ 4부터 시작(77레벨 대부분은 1~3선분 의도 해). 석필/서리/화염 잉크 전부 지원.
- **탐색기**: (a) 격자 스냅 무작위 샘플링 → (b) 상위 해 주변 국소 정련(좌표 ±섭동, 선분 축소)
  2단계. 각 후보를 헤드리스 심으로 T틱(레벨별 상한) 롤아웃하고 승리 판정.
- **산출**: solvable 여부, min_ink(발견 해 최소 잉크), effort(첫 해까지 롤아웃 수),
  해 아카이브(상위 N개 — 카빙·관용도 계산에 재사용).
- **예산**: 레벨당 롤아웃 상한(기본 2,000)·틱 상한(기본 3,600=60초분). 미발견 = "솔버 한계"로
  기록하고 사람 플레이 대기 (불가해 판정 아님 — 탐색 공간이 사람 창의보다 좁다).
- 결정성 유지: 탐색 시드 고정 → 재실행 동일 결과.

### L2. 다차원 난이도 정량화
레벨당 메트릭 벡터:
- `min_ink` / `budget` → **실측 여유율** (K4 실측치)
- `effort` → 탐색 난이도 (사람 체감 시도 수의 프록시)
- `density` → 관용도: 최적해 주변 교란 M개(기본 200) 중 클리어 비율. 낮을수록 정밀 요구
- `complexity` → 해 선분 수·잉크 종류 수·필요 상전이 수·기믹 경유 수 (K3·K5·K6 프록시)
- `timing` → 해의 시간 민감도: 스트로크 착수 틱을 ±지연시켰을 때 클리어 유지 비율 (K7 프록시)
- 합성 D̂(추정 난이도) = 가중 합성 (초기 가중치는 챕터 1~2 실측으로 캘리브레이션,
  LEVELS.md §2.2 D밴드 체감 정의와 회귀 대조)

### L3. 카빙 (수술)
- **예산 카빙**: `optimal_ink := min_ink` 승격 → `ink_budget := min_ink × D밴드 여유율`,
  `star_thresholds` §4 공식(클램프 포함) 재산정. — 백로그 "optimal_ink 전량 추정치" 해소.
- **지형 카빙** (선택 적용): 설계 D̂가 목표 D보다 낮은 레벨에 한해 벽 추가·통로 축소 후보를
  1개씩 적용→L1 재검증→D̂ 재계산의 greedy 루프 (차곡 wall_carver_v2 방식).
  슬롯 불변·핵심 아이디어 불변·teaches 불변이 하드 제약. 자동 커밋 금지 — 제안 diff로 출력.
- **관용도 하한**: density가 교육 레벨(dip)에서 기준 미만이면 역카빙(공간 완화) 플래그.

### L4. 분포 감사
- 77레벨 전수: solvable 100%(또는 솔버 한계 목록), D̂ vs 설계 D 상관, 챕터별 D 곡선
  단조성(작업 레벨 스파이크 5→6→7→7→8→9→10), 여유율 밴드 준수, 관용도 곡선(챕터 진행에
  따라 하강), 이상치(D̂-D 괴리 상위) 리포트.
- 산출: `_workspace/level_lab_audit_<날짜>.md` + 레벨별 메트릭 JSON(`tool/level_lab/out/`).

### L5. 생성 (확장 — 보너스·챕터 5 대비)
- 챕터 템플릿 + 변이(차곡 seed_mutator 방식): 기존 레벨을 시드로 지형·배치·예산 변이 →
  L1~L2 통과 후보를 D bin별 아카이브(MapElites 방식)에 수집.
- 런칭 77레벨은 수제 유지(GDD 원칙). 생성은 보너스 레벨·챕터 5·데일리 퍼즐 후보 공급용.

## 2. 실행 형태

```
dart run tool/level_lab/solve.dart --level assets/levels/level_017.json   # 단일 솔브
dart run tool/level_lab/sweep.dart --all                                  # 77레벨 전수 L1+L2
dart run tool/level_lab/carve.dart --level ... --budget-only              # L3 예산 카빙 제안
dart run tool/level_lab/audit.dart                                        # L4 분포 감사
```

- 병렬: Isolate 풀로 레벨 단위 병렬 (77레벨 × 2,000롤아웃 × ~수백 틱 ≈ 수십 분 예상 — 실측 후 조정).
- 앱 코드 수정 최소화: 솔버는 lib/gameplay(LevelSession)·lib/sim(GameState)을 그대로 소비.
  필요한 훅(헤드리스 스트로크 주입·틱 루프·판정 조회)이 부족하면 lib에 최소 API만 추가.

## 3. 소유권·단계 게이트

| 단계 | 소유 | 게이트 |
|---|---|---|
| L1 솔버 | gameplay-engineer (+ sim 훅 필요 시 sim-engineer) | 챕터 1 11레벨 전부 자동 해 발견 (석필+중력 반전만) |
| L2 메트릭 | gameplay-engineer → level-designer 캘리브레이션 | 챕터 1~2 D̂ vs 설계 D 상관 확인 |
| L3 카빙 | level-designer (제안) + 리더 승인 (JSON 반영) | 재검증 solvable 100% 유지 |
| L4 감사 | game-qa | 리포트 발행 |
| L5 생성 | level-designer | 보너스 레벨 설계 확정 후 |

- 레벨 JSON 자동 수정은 L3 예산 카빙만 허용(공식 기계 적용), 지형 카빙은 제안→사람 승인.
- 본 파이프라인 산출(min_ink 등)은 LEVELS.md §4 "에디터 저작자 플레이 기록"과 동격의
  실측으로 취급하되, 사람 실플레이가 더 낮은 min_ink를 내면 그쪽이 우선.
