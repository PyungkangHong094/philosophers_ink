## 2. 코어 루프 (30초)

1. **관찰** — 방출구(레토르트 목) 위치, 플라스크 위치·요구 조건, 잉크 예산 파악
2. **긋기** — 잉크 선택 후 드로잉. 선이 즉시 물리에 반영됨
3. **변환** — 원소가 흐르고, 녹고, 얼고, 증발하는 연쇄를 지켜봄 (핵심 보상 순간)
4. **채움** — 플라스크 카운트업 → 클리어 → 별점 → 다음 레벨

실패 개념: 명시적 실패 없음. 원소가 화면 밖으로 새어도 계속 방출되므로 언제든 만회 가능.
단, 순수 플라스크 오염 시와 유한 방출 레벨에서 목표 미달 시에는 재시작 버튼으로 리셋한다.

## 3. 원소 체계

### 3.1 물질 테이블

셀 하나 = 물질 ID 1개. 물질별 속성은 중앙 테이블(`materials.dart`)에 정의한다.

| ID | 물질 | 카테고리 | 가열 시 | 냉각 시 | 비고 |
|---|---|---|---|---|---|
| EMPTY | 빈 칸 | — | — | — | |
| WALL | 석필 선 | 정적 | — | — | 플레이어 드로잉 |
| HEAT_LINE | 화염 룬 선 | 정적 | — | — | 인접 셀 가열 |
| COLD_LINE | 서리 룬 선 | 정적 | — | — | 인접 셀 냉각 |
| PRIMA | 프리마 (제1질료) | 입자 | 불변 | 불변 | 튜토리얼·질량 담당, 불활성 |
| ICE | 얼음 | 입자 | → WATER | 불변 | 안식각 낮음 (미끄러워 잘 퍼짐) |
| WATER | 물 | 액체 | → STEAM | → ICE | 시스템의 주인공 |
| STEAM | 증기 | 기체 | 불변 | → WATER | 유일한 상승 물질 |
| ASH | 재 (오염) | 입자 | 불변 | 불변 | 순수 플라스크 오염원 |
| LAVA | 이그니스 (용암) | 액체 | 불변 | → STONE | 챕터 4 해금 |
| STONE | 돌 | 입자 | 불변* | 불변 | 반응 생성물 |

*STONE의 재가열(→LAVA)은 런칭 범위에서 제외. 확장 여지로만 남긴다.

### 3.2 반응 (물질 간 상호작용)

런칭 범위의 반응은 단 하나:

- **LAVA + WATER 접촉 → STONE + STEAM** (접촉한 두 셀이 각각 변환)

이 외의 물질 간 반응은 추가하지 않는다 (스코프 통제).

### 3.3 카테고리별 이동 규칙

| 카테고리 | 이동 우선순위 |
|---|---|
| 입자 | 아래 → 아래대각 (좌우 랜덤) |
| 액체 | 아래 → 아래대각 → 수평 확산 (dispersion 파라미터) |
| 기체 | 위 → 위대각 → 수평 (액체의 상하 미러) |
| 정적 | 이동 없음 |

- 좌우 스캔 방향은 프레임마다 교차하여 편향을 제거한다.
- 중력 반전 시 입자/액체와 기체의 이동 방향이 서로 뒤바뀐다 (규칙 자체를 미러링).

## 4. 잉크 시스템

### 4.1 잉크 3종

| 잉크 | 색 | 효과 |
|---|---|---|
| 석필 | 배경 대비 밝은 톤 | 물리 벽. 원작의 그 선 |
| 화염 룬 | 주홍 | 인접 셀을 매 틱 확률 p_heat로 +1단계 가열 |
| 서리 룬 | 청백 | 인접 셀을 매 틱 확률 p_cold로 −1단계 냉각 |

- 확률적 전이를 쓰는 이유: 즉시 전환보다 자연스럽고(가장자리부터 서서히 녹음), 셀당 온도 값을 저장할 필요가 없어 메모리·연산이 싸다.
- **핵심 창발**: 서리선 위에 고인 물이 얼음 입자가 되어 쌓임 → 더미가 곧 다리·지형. 별도의 정적 얼음 타입 없이 입자 퇴적만으로 "얼려서 짓기"가 성립한다. 화염선으로 다시 녹여 철거 가능.

### 4.2 잉크 예산

- 단위: 래스터라이즈된 셀 수. 레벨별로 잉크 종류당 예산을 지정한다.
- UI: 하단 잉크병 3개 + 잔량 게이지. 예산 0인 잉크는 병 자체를 숨긴다 (튜토리얼 단순화).
- 선 삭제: 그은 선을 탭하면 스트로크 단위로 삭제. **잉크는 반환하지 않는다** (신중한 드로잉 유도).
- 예산 부족 시: **부분 배치(잔량 cap)** — 드래그 중 실시간 차감되며, 잔량이 다하면 선이 그 지점에서
  멈춘다. 전부-아니면-전무(스트로크 통째 거부) 방식은 쓰지 않는다 (원작의 조작 손맛 계승,
  드래그 중 무반응은 입력 신뢰를 해침). [2026-07-16 M1에서 확정]

### 4.3 해금 순서

| 시점 | 해금 | 근거 |
|---|---|---|
| 챕터 1 | 석필만 | 원작과 동일한 온보딩 |
| 챕터 2 | 서리 룬 | "물이 얼어 다리가 되는" 가시적 건축이 첫 마법 경험으로 가장 직관적 |
| 챕터 3 | 화염 룬 | 증발·응결은 한 단계 추상적이므로 후순위 |

## 5. 승리 조건과 별점

### 5.1 플라스크 요구 조건

| 표기 | 조건 |
|---|---|
| 숫자만 | 해당 개수만큼 어떤 물질이든 카운트 |
| 물질 색 플라스크 | 해당 물질만 카운트 (타 물질은 통과·소멸) |
| 상태 아이콘 (물방울/눈꽃/김) | 해당 상(액체/고체/기체)으로 착수해야 카운트 |
| ❗ (순수) | ASH가 1개라도 들어가면 실패 → 재시작 |

각주: 카운트 대상 "물질"은 방출·상전이로 생긴 동적 물질(입자/액체/기체)에 한한다.
플레이어가 잉크로 그린 정적 셀(석필·화염/서리 룬 선)과 벽은 플라스크 내용물이 아니며
카운트·소비되지 않는다. (2026-07-18 — 플라스크 내 드로잉 즉시 클리어 익스플로잇 봉쇄)

넘침 규칙 (2026-07-19 사용자 디렉션): **목표를 채운 플라스크는 더 이상 소비하지 않는다.**
이후 도달하는 물질은 소비되지 않고 물리 그대로 쌓여 넘친다 — "다 찼다"가 시각·물리로
즉시 읽히고, 다중 플라스크 레벨에서 남은 흐름을 다른 곳으로 돌려야 할 이유가 생긴다.
(순수 플라스크의 ASH 오염 판정은 목표 달성 전까지만 유효 — 채운 뒤 넘치는 ASH는 무해.)

상태 지정이 만드는 퍼즐: 증기로만 도달 가능한 높은 플라스크에 "액체" 조건이 붙으면,
아래서 증발 → 상승 → 위에서 응결의 사이클을 강제하게 된다.

### 5.2 방출

- 기본: 무한 방출 (원작 계승, 만회 가능성 보장)
- 레벨 옵션: 유한 방출 (총량 지정). 고난도 레벨의 긴장 장치로만 사용하고 챕터 3 이후에 등장시킨다.

### 5.3 별점

| 별 | 조건 |
|---|---|
| ★ | 클리어 |
| ★★ | 잉크 잔량 합계가 레벨 임계 A% 이상 |
| ★★★ | 잉크 잔량 합계가 레벨 임계 B% 이상 (A < B, 레벨별 에디터에서 설정) |

별점은 재도전 동기 + 챕터 보너스 레벨 해금 조건으로 사용한다.

## 6. 환경 기믹 (레벨 요소)

| 기믹 | 동작 | 원작 대응 |
|---|---|---|
| 변성 게이트 | 통과한 물질을 다른 물질로 변환 (예: PRIMA→WATER) | 색 필터의 고도화 |
| 중력 반전 버튼 | 탭 시 전체 중력 반전. 기체는 가라앉고 입자는 떠오름 | 계승 + 상전이와 곱셈 |
| 포탈 | 페어 박스 간 물질 순간이동 | 계승 |
| 온도 존 | 레벨 고정 화로/빙결 지대. 잉크 해금 전 상전이 교육 담당 | 신규 |
| 재 방출구 | ASH를 함께 쏟아내는 방출구. 순수 플라스크와 조합 | ! 컵의 고도화 |

**설계 원칙 — 곱셈**: 기믹을 단독으로 소비하지 않는다. 새 기믹 등장 레벨(교육) 직후,
반드시 기존 기믹·잉크·상태와 곱해진 레벨을 배치한다. 조합 공간 = 기믹 5 × 상 3 × 잉크 3.

## 7. 진행 구조

### 7.1 챕터 — 연금술 대업 4단계 × 7대 작업 (77레벨)

77 = 7 × 11. 연금술의 7대 작업(Seven Operations)이 11레벨 간격의 마일스톤 시험이 되고,
챕터 경계는 11의 배수에 정렬된다. 레벨별 상세는 부속 문서 `PHILOSOPHERS_INK_LEVELS.md`가 정의한다.

| 챕터 | 이름 | 배경 팔레트 | 신규 요소 | 레벨 | 작업 레벨 |
|---|---|---|---|---|---|
| 1 | 니그레도 (흑화) | 암갈·흑 (#1D1418 계열) | 석필, PRIMA, 중력 반전 | 1–11 | 11 하소 |
| 2 | 알베도 (백화) | 은백 (#E9E5DB 계열) | 서리 룬, WATER/ICE, 온도 존, 상태 플라스크, 포탈, 변성 게이트 | 12–33 | 22 용해 · 33 분리 |
| 3 | 키트리니타스 (황화) | 금 (#D9A62E 계열) | 화염 룬, STEAM, 재/순수, 유한 방출 | 34–55 | 44 결합 · 55 발효 |
| 4 | 루베도 (적화) | 진홍 (#8E1F2F 계열) | LAVA, STONE, 용암+물 반응 | 56–77 | 66 증류 · 77 응고 |

- 각 레벨의 배경은 챕터 기준색에서 명도·채도만 변주한다 (원작의 "레벨마다 단색" 미학 계승).
- 튜토리얼 챕터가 가장 짧은 것(11레벨)은 의도 — 첫 마법(서리 룬, 레벨 17)까지 빨리 데려간다.
- 작업(OPERATIO) 레벨: 직전 10레벨의 종합시험. 레벨 셀 골드 링, 전용 스팅어, 클리어 시 작업 인장 획득. 힌트 비활성.
- 런칭 77레벨. 이후 챕터 5 "퀸타 에센치아(제5원소)"(엔드게임)를 무료 업데이트로 예약.
- 별점 누적으로 챕터당 보너스 레벨 2~3개 해금.

### 7.2 튜토리얼 원칙

- 텍스트 최소화. 원작처럼 화면 내 짧은 문구 1~2줄만.
- 새 요소는 "그것만으로 클리어되는 레벨"에서 등장 → 다음 레벨에서 곱셈.

## 8. 아트 디렉션

### 8.1 스타일

- 배경: 챕터 팔레트 flat 단색. 그라데이션·텍스처 없음.
- 원소: **둥근 잉크 방울 파티클** (셀 = 화면 픽셀 정수배 격자 유지, 동적 물질은 원형 점,
  정적 잉크 선·벽은 각진 사각 — 선의 명료함 유지). 2026-07-19 개정: 사용자 디렉션으로
  픽셀 사각 도트 → 잉크 방울 원형으로 전환. "잉크" 픽션 정합.
- 오브젝트: 볼드한 단색 실루엣. 방출구는 레토르트(증류기) 목, 컵은 플라스크·비커.
- 잉크 선: 손그림 질감. 화염/서리 룬 선 위에 미세한 룬 문양이 흐르는 애니메이션 (폴리시 단계).

### 8.2 제작 규칙 (game-art 원칙 적용)

- **1x 제작, 정수배 확대**: 모든 도트 에셋은 논리 해상도로 제작하고 `FilterQuality.none`으로만 확대. 축소 금지.
- **실루엣 가독성**: 모든 오브젝트는 단색 실루엣만으로 식별 가능해야 한다. 플레이 거리(폰을 든 팔 길이)에서 테스트.
- **가독성은 명도 대비**: 물질 색은 색상(hue)이 아니라 배경 대비 명도로 우선 구분한다 (색약 대응). 같은 배경에서 물질 간 명도 차 최소 20% 확보.
- 에셋 네이밍: `spr_flask_pure_idle.png` 식의 `[type]_[object]_[variant]_[state]` 규칙.

### 8.3 UI

- 인게임 UI는 하단 잉크 팔레트(병 3개 + 게이지)와 우상단 일시정지·재시작뿐.
- 레벨 번호·플라스크 카운트는 원작처럼 배경 위 대형 타이포로 통합.

### 8.4 셸 디자인 시스템 (아웃게임)

인게임(8.1)과 셸(타이틀·챕터/레벨 선택·설정·결과·스토어 에셋)은 역할을 분리한다.
**셸은 조용하게, 인게임은 화려하게** — 셸의 무채색이 레벨 진입 순간 챕터 단색을 터뜨리는 대비 구조.
인터랙티브 목업: `philosophers_ink_shell_mockup.html` (이 문서의 시각적 부속물, 스펙 충돌 시 본 문서가 우선).

#### 8.4.1 철학

- 레퍼런스 톤: 람보르기니 웹 디자인 언어 — 트루 블랙 표면, 골드 악센트, 드라마틱한 대형 타이포.
- 픽션 정합: 블랙 = 니그레도(대업의 출발점), 골드 = 연금술의 최종 목표(금). 셸은 연금술사의 서재다.
- 셸의 모든 전환은 "의식(ritual)"처럼 — 짧고 가볍게가 아니라, 무게감 있는 duration과 강한 ease로.
- **골드 희소성 원칙: 화면당 골드 요소 1~2개.** 골드가 흔해지는 순간 금이 아니게 된다.

#### 8.4.2 컬러 토큰

```
// 표면 (블랙은 순수 무채가 아니라 그을린 웜 블랙 — 니그레도의 탄 질감)
--black-0:   #050505   // 최심부 (타이틀·결과 배경)
--black-1:   #0A0A09   // 기본 캔버스
--black-2:   #131311   // 카드·패널
--black-3:   #1D1C19   // 엘리베이티드·프레스
--hairline:  #29271F   // 보더 (골드 기운이 도는 헤어라인)

// 골드 (악센트 1계열만)
--gold:      #C9A227   // 기본 — CTA·선택·별
--gold-hi:   #E8C95A   // 글로우 코어·프레스 하이라이트
--gold-deep: #8F7118   // 섀도·비활성 골드

// 텍스트 (순백 금지 — 양피지 뉘앙스)
--parchment: #F2EDDF   // 주 텍스트
--text-2:    #9C968A   // 보조
--text-3:    #5E5A50   // 비활성·힌트

// 유채색은 챕터 스와치 4색(7.1)만 셸에 등장할 수 있다
```

의미 규칙: 골드 = "가치·현재·달성"(CTA, 현재 레벨, 획득 별). 상태 표현(잠김/비활성)은 색이 아니라
명도(text-3)와 보더 유무로 해결한다. 니그레도 스와치는 블랙 위에서 거의 안 보이는데, 이는 의도다
(흑화는 어둠 속의 어둠) — 식별은 헤어라인과 텍스트가 담당한다.

#### 8.4.3 타이포그래피

한/영 이중 전략을 쓴다. 한글에는 대문자가 없으므로 "드라마틱 대문자"의 역할을 라틴이 전담한다.

| 역할 | 서체 | 용례 |
|---|---|---|
| Display (라틴) | Anton 계열 컨덴스드 헤비, 대문자 전용 | 챕터명(NIGREDO), 레벨 번호, 타이틀 로고 |
| 본문·한글 헤딩 | Pretendard (Black~Regular) | 한글 전반. 큰 한글은 900 웨이트 |
| 유틸리티 | 본문 서체 소형 + 자간 +0.25~0.35em | 아이브로우 라벨 (MAGNUM OPUS 등) |

- 스케일: Display XL(레벨 번호, 화면 폭의 15~20vw) / L(챕터명 34~64px) / 한글 헤딩 44px /
  본문 15px / 캡션 12px / 아이브로우 11px.
- 숫자는 tabular figures 고정 (카운트·진행도 정렬 유지).
- 라틴 대문자 자간 +0.02~0.05em, 한글 대형은 자간 −1~−2%.

#### 8.4.4 화면별 스펙

| 화면 | 스펙 |
|---|---|
| 타이틀 | black-0. 중앙 골드 플라스크 라인아트 — 내부에서 골드 입자가 미세하게 상승, 글로우가 4초 주기로 호흡. 로고는 라틴 Display 2행. 하단 "화면을 터치하여 시작" 펄스. 골드 요소: 플라스크 1개 |
| 챕터 선택 | 세로 카드 4장. 카드 = black-2 + 헤어라인 + 좌측 3px 챕터색 스파인. 라틴 챕터명 대형 + 한글명·레벨 범위 캡션 + 우측 진행도(tabular)·획득 별. 잠금 카드는 text-3 + "N장 완료 시 해금"(방향 제시, 감성 문구 금지). 골드 요소: 별 카운트, 현재 챕터 보더 |
| 레벨 선택 | 상단 라틴 챕터명 대형 + 한글 부제. 5열 그리드 셀(정방형, black-2, 헤어라인). 클리어 셀 = 번호 parchment + 하단 소형 골드 별 1~3. 현재 셀 = 골드 보더 + 글로우. 잠금 = text-3. 골드 요소: 현재 셀, 획득 별 |
| 클리어 | black-0 오버레이. 아이브로우(챕터·레벨) → 한글 Display "정제 완료" → 골드 별 3개 순차 스탬프(햅틱 동기) → 잉크 잔량 게이지(골드 필) → CTA [다음 레벨](골드 필) / [다시 정제](고스트) |
| 일시정지 | 배경을 black-0 오버레이 90%로 죽임(챕터색 차단이 목적, 블러 금지 — 성능). 세로 텍스트 버튼 3개(계속/재시작/나가기), 고스트 스타일 |
| 설정 | 리스트 + 헤어라인 구분. 토글 온 상태만 골드 |
| 스토어 에셋 | 아이콘 = black-0 + 골드 플라스크 실루엣(라인아트 아님, 면 실루엣 — 소형 가독성). 스크린샷 = 블랙 프레임 + 상단 라틴 대형 카피 + 인게임 챕터색 화면 |

#### 8.4.5 모션

- **시그니처 — 잉크 플러드**: 레벨 셀 탭 → 탭 좌표를 기점으로 해당 챕터색 원이 방사형으로 확장되어
  화면을 삼키고 그대로 인게임 배경이 됨 (650ms, easeInOutQuart). 셸→인게임 전환의 유일한 유채색 순간.
  클리어 후 복귀는 역방향(챕터색이 탭 지점으로 수축하며 블랙 복귀).
- duration 토큰: fast 120ms(프레스 피드백) / base 240ms(화면 페이드) / ritual 650ms(플러드, 별 스탬프 시퀀스).
- 별 스탬프: scale 1.6→1.0 오버슛 + 글로우, 250ms 간격 스태거, 각 스탬프에 햅틱(light impact).
- 타이틀 플라스크 호흡: 글로우 강도 사인파 4s 주기. 입자 상승 6~8개, 불규칙 딜레이.
- reduced motion 설정 시: 플러드는 즉시 컷, 호흡·펄스 정지, 스탬프는 페이드로 대체.

#### 8.4.7 품질 바닥

- 대비: gold on black-0 ≈ 7:1, parchment on black-0 ≈ 15:1 — 본문 AA 이상 유지. text-3는 비활성 전용.

- reduced motion·햅틱 오프 설정 제공.

## 9. 사운드 디자인

### 9.1 방향

핵심은 파티클 ASMR. 워리스톤에서 확보한 flutter_soloud 실시간 변조 노하우를 재활용한다.

### 9.2 레이어 구조 (game-audio 원칙 적용)

| 레이어 | 내용 | 구현 |
|---|---|---|
| 파티클 그레인 | 입자 낙하·퇴적의 지속음 | 활성 파티클 밀도 → 볼륨·피치 실시간 변조 (수직 적응형) |
| 물질 앰비언트 | 물 흐름, 증기 히스 | 물질별 활성 셀 수에 비례 |
| 이벤트 SFX | 결빙 crackle, 증발 puff, 플라스크 착수 틱, 클리어 스팅어 | **변주 3~5개씩** 제작해 반복감 제거 |
| BGM | 챕터별 미니멀 앰비언트 루프 | 클리어 시 덕킹 + 스팅어 |

- 믹스 계층: 이벤트 SFX 기준 0dB > 파티클 그레인 −6dB > BGM −9~−12dB.
- 플라스크 착수음은 카운트업과 동기화 — 원작의 만족감 핵심이므로 최우선 폴리시 대상.
- 오디오 예산: 총 30MB 이하 (모바일 캐주얼 기준).
- 음소거 토글 필수 (설정 + 인게임 접근).

## 10. 기술 아키텍처

### 10.1 스택

- Flutter, 게임 엔진 없이 raw `CustomPainter` + `Ticker`. Flame 미사용 (단일 풀스크린 시뮬 캔버스에는 오버헤드).
- 시뮬레이션: falling-sand 셀룰러 오토마타(CA), 메인 스레드. M0 프로파일링에서 예산 초과 시에만 Isolate 이전.

### 10.2 시뮬 코어

- 논리 그리드: 세로형 약 160×320 (기기 비율 따라 세로 가변, ≈51k 셀).
- 표현: `Uint8List` 1개, 셀당 물질 ID 1바이트. 물질 속성(카테고리·전이·색)은 중앙 테이블 참조.
- 틱: 60Hz 고정, accumulator로 프레임과 분리.
- **결정성**: 시드 고정 RNG 단일 인스턴스. 같은 레벨·같은 입력이면 같은 결과 (재시작 동일성, 향후 리플레이·힌트 검증 기반).
- 화염/서리 선: 인접 4방향 셀을 매 틱 확률 p로 전이. p는 상수 테이블에서 관리.

### 10.3 렌더링 파이프라인

그리드 `Uint8List` → 팔레트 룩업으로 RGBA 버퍼 → `ui.ImmutableBuffer` + `ImageDescriptor.raw` → `drawImageRect` (FilterQuality.none, 정수배).

프레임 예산 (16.6ms): 시뮬 ~3ms + 버퍼 변환 ~1ms + 페인트 ~2ms. M0에서 실측 검증.

### 10.4 드로잉 입력

- 터치 스트로크 → 브레젠험으로 그리드 래스터라이즈, 선 두께 2셀.
- 잉크 차감 = 래스터라이즈된 셀 수.
- 스트로크는 ID를 유지 (탭 삭제 단위). 삭제 시 해당 셀을 EMPTY로 복원.

### 10.5 코드 구조 (game-architecture 원칙 적용)

```
lib/
├── core/       # game_loop, constants, rng, game_state
├── sim/        # grid, materials(물질 테이블), rules(카테고리별 이동·전이)
├── render/     # world_painter, palette
├── level/      # level_model, loader, validator, editor(디버그 전용)
├── gameplay/   # ink_budget, flask, gimmicks, star_rating
├── ui/         # palette_bar, hud, level_select
└── meta/       # progress, settings, ads, iap, analytics
```

- **상수 중앙화**: 모든 밸런스 값(전이 확률, dispersion, 방출 속도 기본값, 별점 임계 기본값)은 `constants.dart` 한 곳에. 게임 로직에 매직 넘버 금지.
- **재시작 안전**: `GameState.reset()`이 그리드·잉크·플라스크·RNG 시드를 완전 초기화. 3회 연속 재시작 시 동일 동작을 테스트로 보장.
- 시뮬(sim/)은 Flutter 위젯에 의존하지 않는 순수 Dart로 유지 → 단위 테스트 가능.

### 10.6 레벨 데이터

- 포맷: JSON. 방출구(위치·물질·속도·총량), 플라스크(위치·목표·조건), 기믹 배치, 잉크 예산, 배경색, 별점 임계.
- **인앱 레벨 에디터 (디버그 빌드 전용)**: 폰에서 직접 배치·테스트·JSON 익스포트. 60레벨 제작 속도의 핵심 투자.
- 배포: 레벨 JSON은 에셋 번들 + Shorebird OTA로 심사 없이 추가·밸런스 패치.

| "클론" 인식 | 상전이·잉크·연금술 픽션이 차별화 축. 스토어 문구에서 원작 언급 금지, 형태 언어(쉼표 방출구)도 레토르트로 재해석 |
| 액체 거동 품질 | 액체 dispersion 튜닝이 손맛의 절반. M1에서 파라미터 노출 후 체감 튜닝 기간 확보 |
| 레벨 제작 병목 | 인앱 에디터 (M2) 선투자로 해소 |

오픈 이슈 (구현 중 결정):

1. 화염·서리선의 전이 확률 p 기본값 — M1 체감 튜닝
2. 얼음 안식각 파라미터 (미끄러움 정도) — M1 체감 튜닝
3. 힌트용 정답 스트로크 데이터를 에디터에서 함께 저장할지, 별도 제작할지 — M2에서 결정
