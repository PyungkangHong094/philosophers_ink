# QA 검증 프로토콜

판정 기준의 원천: `docs/PHILOSOPHERS_INK_DESIGN.md` (특히 10장 예산·14장 체크리스트),
`docs/PHILOSOPHERS_INK_LEVELS.md` 8장. 기준이 문서에 없으면 "기준 미정의"로 분리 보고.

## 기본 실행 순서 (모듈 검증 요청 수신 시)

1. `dart analyze` — 정적 에러 0 확인.
2. `flutter test <대상>` — 기존 테스트 회귀 확인.
3. 이전 QA 리포트(`_workspace/qa_*.md`) 확인 — 과거 실패 항목 회귀 재검 최우선.
4. 아래 경계면 목록 중 해당 항목 교차 검증.
5. 리포트를 `_workspace/qa_{모듈}_{YYYYMMDD}.md`로 저장, 결함은 소유 에이전트에 통지.

## 경계면 교차 검증 목록

경계면 검증 = 양쪽 코드를 **동시에 읽고** 계약(shape·단위·타이밍)을 비교하는 것.

| 경계면 | 확인 사항 |
|---|---|
| sim ↔ gameplay | 기믹이 호출하는 sim API 시그니처 일치. 물질 ID enum 값 공유 (중복 정의 금지) |
| gameplay ↔ level JSON | 로더가 파싱하는 필드명·타입 = 스키마 문서 = 실제 JSON 파일 3자 일치 |
| gameplay ↔ ui | 플라스크 카운트·별점 이벤트 shape, 진행도 저장 API 계약 |
| render ↔ sim | 팔레트 인덱스 = 물질 ID 순서 일치 (materials 테이블 변경 시 팔레트 회귀) |
| ui ↔ tokens | 위젯 코드 내 하드코딩 hex 검색 (`grep -rn "Color(0x" lib/ui/ lib/meta/` — tokens.dart 외 0건) |

## 정기 감사 항목

- **결정성**: 같은 레벨·같은 스트로크 입력 → N틱 후 그리드 해시 3회 동일. 재시작 3회 동일 동작.
- **매직 넘버**: `lib/sim|gameplay/` 내 숫자 리터럴 검색 — constants.dart·레벨 JSON 외 출처 0개 (좌표 오프셋 ±1 등 알고리즘 상수는 예외, 밸런스 값은 무관용).
- **성능**: 디버그 계측(Stopwatch)으로 시뮬 3ms/변환 1ms/페인트 2ms 예산 확인. 프로파일 빌드 기준.
- **셸 품질 바닥**: 터치 타겟 44px+(레벨 셀 56px), 골드 요소 화면당 1~2개(작업 레벨 골드 링은 예외),
  reduced motion 분기 존재.
- **레벨 정합성**: 해금 순서 위반(챕터보다 이른 요소 사용), 슬롯 규칙, 작업 레벨 D 단조 증가.

## 리포트 형식

```markdown
# QA: {모듈} — {날짜}
## 요약: PASS n / FAIL n / 권고 n / 기준미정의 n
## FAIL (P0/P1)
- [P0] {제목} — 재현: ... / 근거: GDD §x.x "..." / 소유: {agent}
## 권고
## 회귀 확인 (이전 리포트 대비)
```

P0 = 빌드 깨짐·결정성 위반·데이터 손상. P1 = 스펙 위반. 권고 = 문서 근거 없는 개선 제안.

## 출시 전 체크리스트

GDD 14장 체크리스트 전 항목을 실행 검증으로 확인 (읽기 검토로 대체 금지).
