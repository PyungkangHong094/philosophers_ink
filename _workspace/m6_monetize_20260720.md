## 구현 요약

### lib/monetize/ (신설)
- `monetize_constants.dart` — 빈도 정책(`AdsPolicy`: 3클리어 간격·90초 쿨다운·첫 세션 유예),
  광고 단위(`AdUnits`: 현재 전부 Google 공식 **테스트 ID**), IAP 상품(`IapProducts.removeAds`).
- `interstitial_cadence.dart` — 전면광고 빈도 게이트(순수 로직, SDK 비의존 → 단위 테스트).
- `ads_service.dart` — `AdsService`(인터페이스) / `StubAdsService`(무광고) / `GoogleAdsService`
  (google_mobile_ads 실구현: 전면·리워드 프리로드·소진 후 재로드). `AdsService.create`가
  kDebugMode면 스텁 선택.
- `iap_service.dart` — `IapService` / `StubIapService`(스토어 미연결 우아한 실패) /
  `StoreIapService`(in_app_purchase 실구현: 구매·복원·purchaseStream 처리).
- `monetization.dart` — `Monetization`(ChangeNotifier 파사드): 광고+IAP 조립, 광고 제거
  소유(`adsRemoved`)를 `ProgressStore`에 영속, 구매/복원 그랜트 시 광고 게이팅 전파.

### 배선
- `InkServices`에 `monetization` 주입(부트스트랩에서 `Monetization.create(store)` + `init()`).
  실패해도 게임을 막지 않음.
- **전면광고**: 레벨 클리어 → "다음 레벨" 이동 사이(`level_select_screen` onNext)에서
  `monetization.onLevelCleared()` — 빈도 정책 통과 + 미제거 시에만 노출.
- **리워드 힌트**: 플레이 상단 버튼(전구) + 일시정지 오버레이 "힌트 보기" → 리워드 성공 시
  `hint_stroke` 폴리라인을 **중립 parchment 고스트 라인**으로 오버레이(골드 아님).
  노출 조건: `hint_stroke` 2점 이상 && 비-OPERATIO(11배수) 레벨. (미준비 시 -어요체 안내.)
- **IAP**: 설정 화면에 "광고 제거"(구매) / "구매 복원" 진입점. 구매 완료면 "광고 제거 · 완료"
  상태 행으로 전환. 결과 피드백은 -어요체 스낵바. 설정 리스트를 스크롤 가능하게 전환(오버플로 방지).
- `ProgressStore` — `ink.adsRemoved.v1` bool 키 영속.

### 네이티브 / 앱 아이덴티티
- Android: `AndroidManifest.xml`에 `com.google.android.gms.ads.APPLICATION_ID` meta-data(테스트 앱 ID).
- iOS: `Info.plist`에 `GADApplicationIdentifier`(테스트 앱 ID) + `SKAdNetworkItems`(Google 식별자).
- 아이콘: `flutter_launcher_icons`로 `assets/icon/app_icon.png`(1024) → Android/iOS 배선.
- 스플래시: `flutter_native_splash`로 `#050505` 배경 + `splash_logo.png` 네이티브 스플래시(android_12 포함).
