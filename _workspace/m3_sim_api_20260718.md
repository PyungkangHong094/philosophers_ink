# M3 시뮬 기믹 API 계약 (sim → gameplay)

작성: sim-m3 / 2026-07-18 · 대상: gameplay-engineer
범위: `lib/sim/gimmicks.dart`, `lib/sim/rules.dart`, `lib/core/game_state.dart` (시뮬 측만)

gameplay는 레벨 JSON을 파싱해 이 타입들을 만들고 `GameState`에 주입한다.
sim은 매 틱 결정적으로 적용한다(RNG 미사용, 결정성 로그 안정).

---

## 1. 좌표계 규약

- 셀 (x, y)의 선형 인덱스 = `y * gridWidth + x` (row-major, `Grid`와 동일).
- 그리드 크기: `SimConstants.gridWidth`(=160) × `SimConstants.gridHeight`(=320).
- 물질 ID는 `Material.<name>.index` (예: `Material.water.index`). GDD 3.1 순서 고정.

---

## 2. 변성 게이트 — `TransmutationGate`

존 안의 물질을 다른 물질로 변환. **이동 전** 패스에서 적용된다.

```dart
TransmutationGate.rect({
  required int gridWidth,   // = SimConstants.gridWidth
  required int x, y, width, height,   // 게이트 존 (직사각형)
  required int toMaterial,  // 변환 결과 물질 ID
  int? fromMaterial,        // 변환 대상. null이면 "모든 이동 물질"
})
```

의미론:
- `fromMaterial` 지정 → 그 물질만 `toMaterial`로 변환. 예: `fromMaterial: prima, toMaterial: water`.
- `fromMaterial: null` → 존 안의 **이동 물질(입자/액체/기체) 전부**를 변환. 벽·룬 선(정적)·EMPTY는 보존.
- 임의 형상이 필요하면 `TransmutationGate(cellIndices: [...], toMaterial: ..., fromMaterial: ...)` 직접 생성.

배치 팁: 게이트를 **채널 폭 전체를 덮는 1셀 두께 가로 띠**로 두면, 수직 이동
(입자 1칸/틱)이 건너뛰지 못해 통과 물질이 확실히 변환된다. 여러 매핑이 필요하면
같은 존에 게이트 여러 개를 겹쳐 둔다(리스트 순서대로 적용).

---

## 3. 포탈 — `Portal`

입구 셀의 이동 물질을 출구 셀로 순간이동(일방향 A→B). **이동 후** 패스에서 적용.

```dart
Portal.rects({
  required int gridWidth,
  required int entryX, entryY,   // 입구 박스 좌상단
  required int exitX, exitY,     // 출구 박스 좌상단
  required int width, height,    // 입·출구 공통 크기
})
```

의미론:
- `entryCells[i] → exitCells[i]` 1:1 매핑 (두 박스를 row-major로 짝지음).
- 이동 물질(입자/액체/기체)만 이동. 정적·EMPTY는 무시.
- **막힘 처리**: 출구 셀이 비어있지 않으면 입구에서 대기(GDD 6).
- **연쇄 방지**: 한 틱에 한 번만 이동. 텔레포트로 막 채워진 셀은 같은 틱에 다시 입구로 소비되지 않는다.
- 양방향이 필요하면 방향을 뒤집은 `Portal`을 하나 더 추가한다.
- 임의 형상은 `Portal(entryCells: [...], exitCells: [...])` 직접 생성 (길이 동일 필수).

---

## 4. 중력 반전 — 런타임 토글

전역 중력 반전 버튼(GDD 6). 존 단위가 아니라 **전역**이며, 입력 이벤트다.

```dart
gameState.setGravityInverted(bool inverted);  // 버튼 탭이 호출
bool inverted = gameState.gravityInverted;     // 현재 상태
```

의미론:
- 반전 시 입자·액체는 위로, 기체는 아래로 이동한다(규칙 자체를 미러링, GDD 3.3).
- `GameState.reset()`이 기본(아래)으로 되돌린다.
- **결정성 계약**: 이 토글은 입력 시퀀스의 일부다. 결정성/리플레이 로그를 만들 때
  스트로크와 동일하게 "몇 번째 틱에 어떤 값으로 토글했는지"를 기록해야 재현된다.
  (스트로크처럼 sim 내부에서 자동 기록하지 않는다 — 호출 시점은 gameplay 소유.)

---

## 5. 온도 존 — `TemperatureZone`

레벨 고정 화로/빙결 지대(GDD 6). 잉크 룬 없이 존 안의 셀을 매 틱 가열/냉각한다.
룬 선과 같은 상전이 패스에서 처리(이동 전).

```dart
TemperatureZone.rect({
  required int gridWidth,   // = SimConstants.gridWidth
  required int x, y, width, height,        // 존 (직사각형)
  required TemperatureZoneKind kind,       // .heat(화로) | .cool(빙결)
  double? probability,      // 셀당 매 틱 ±1단계 전이 확률. null이면 룬과 동일(pHeat/pCold)
})
```

의미론:
- 룬 선은 **인접 4방향**으로 복사하지만, 온도 존은 **존 안의 셀 자신**에 직접 적용한다
  (존 전체가 온도장). ICE→WATER→STEAM(가열) / STEAM→WATER→ICE(냉각) 사슬을 그대로 탄다.
- `probability`는 존별 강도 노브. 생략 시 `SimConstants.pHeat`/`pCold`(현재 0.12, 룬과 동일)로 해석.
  값을 넘기면 그 존만 더 세게/약하게(0이면 전이 없음). 코드에 매직 넘버 없음 — 기본값은 상수에서 온다.
- 전이 불가 물질(벽·EMPTY·불활성 PRIMA/ASH/STONE)은 **RNG를 소비하지 않는다**(결정성 안정, 룬과 동일 규칙).
- 임의 형상은 `TemperatureZone(cellIndices: [...], kind: ..., probability: ...)` 직접 생성.
- **온도 존만 RNG를 소비**한다(확률 전이). 게이트·포탈·중력은 결정적(RNG 미사용).

배치 팁: 존은 룬보다 강하다(존 안 모든 셀이 매 틱 판정 vs 룬은 인접 셀만). 화로/빙결 지대의
"체류 시간 = 상전이량" 감각을 위해 존을 통과 경로에 충분히 크게 잡는다.

## 6. 재/순수 (ASH) — 기존 반영, 신규 없음

`재 방출구`는 M1의 `EmitterConfig.ashRatio`로 이미 구현됨. 중복 구현 불필요.

```dart
EmitterConfig(
  x: ..., y: ..., width: ..., materialId: Material.water.index,
  ashRatio: 0.3,   // 0~1: 방출 셀이 이 확률로 ASH로 치환 (결정성 RNG)
);
```

- ASH는 입자로 낙하·퇴적하며 가열·냉각에 불변(오염원). 순수(❗) 플라스크 판정은 gameplay 소유.
- `ashRatio: 0`이면 RNG를 소비하지 않는다.

## 7. GameState 주입 (레벨 로더 배선점)

```dart
final game = GameState(
  seed: ...,                       // 레벨 시드 (없으면 SimConstants.defaultSeed)
  emitters: [ EmitterConfig(...) ],
  gates:            [ TransmutationGate.rect(...) ],  // 없으면 생략 (기본 const [])
  portals:          [ Portal.rects(...) ],            // 없으면 생략 (기본 const [])
  temperatureZones: [ TemperatureZone.rect(...) ],    // 없으면 생략 (기본 const [])
);
```

- `gates`·`portals`·`temperatureZones`는 레벨 데이터(불변).
  `GameState.gates` / `.portals` / `.temperatureZones`로 읽기 가능.
- 틱 순서(고정, 결정적): **방출 → 상전이(룬 → 온도 존) → 변성 게이트 → 이동 → 포탈**.
- `GameState.reset()`이 그리드·RNG·규칙·중력·방출구 잔량을 완전 초기화 → 재시작 3회 동일 해시 보장.

## 9. 상전이 관찰 콜백 (M5 폴리시 — shell-ui SFX/VFX용)

sim이 상전이가 일어난 지점을 알려주는 옵셔널 콜백. shell-ui가 결빙 crackle·증발 puff 등
소리/이펙트를 여기에 연결한다.

```dart
// 타입 (lib/sim/rules.dart)
typedef PhaseChangeCallback = void Function(int materialFrom, int materialTo, int x, int y);

// 배선 (GameState 또는 LevelSession 패스스루 경유)
gameState.onPhaseChange = (from, to, x, y) { /* SFX 트리거 */ };
session.onPhaseChange   = (from, to, x, y) { /* LevelSession 패스스루 (== game.onPhaseChange) */ };
gameState.onPhaseChange = null; // 해제 (기본값)
```

- **파라미터**: `materialFrom`=전이 **전** 물질 ID, `materialTo`=전이 **후** 물질 ID,
  `(x,y)`=셀 좌표. (from,to) 쌍으로 이벤트 종류를 구분한다.
- **발생 지점(3곳)**: 화염/서리 룬 선, 온도 존, LAVA+WATER 반응.
  게이트 변성·이동은 상전이가 아니므로 **발생하지 않는다**.
- **SFX 매핑 예** (from→to):
  - `WATER→ICE` → 결빙 crackle
  - `WATER→STEAM` → 증발 puff
  - `ICE→WATER` 얼음 녹음 / `STEAM→WATER` 증기 응결
  - `LAVA→STONE` 용암 굳음 / `LAVA→STONE`+`WATER→STEAM`(반응) 용암+물 치익
- **결정성 계약**: 콜백은 **관찰 전용** — RNG·그리드 상태를 건드리면 안 된다. 호출 순서는
  결정적(고정 스캔 순서). 관찰 콜백을 달아도 그리드 해시는 null일 때와 동일(테스트로 잠금).
- **비용**: null이면 상전이 지점에서 null 체크 1회뿐 → 성능 회귀 0(벤치 확인: 상전이 대량
  시나리오 활성셀 ~6.9k에서 틱 min ~1.6ms, 예산 3ms).
- **수명**: `reset()`은 이 배선을 지우지 않는다(재시작해도 SFX 연결 유지).

## 10. 미해결 / 인계 사항

- 중력 토글의 입력 로그 기록은 gameplay 소유(4절 결정성 계약 참고).
- 게이트 `fromMaterial: null`의 "이동 물질 전부" 판정은 `isMobile(id)` (gimmicks.dart)와 동일 규칙.
- 온도 존 `probability`를 레벨 JSON에서 넘길 경우, 그 값은 레벨 데이터(밸런스)로 관리(코드 상수 아님).
