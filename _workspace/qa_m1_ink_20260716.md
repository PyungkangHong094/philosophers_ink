### [2a] 차감 = 배치 셀 수 — PASS
GDD 4.2 "단위: 래스터라이즈된 셀 수" 부합. 예산 회계에 두 청구 프리미티브 제공:
- `tryCharge(type, cells)`: 프리뷰 셀 수 기반 사전 all-or-nothing 차감(정확히 cells 차감).
- `chargeAvailable(type, cells)` / 컨트롤러 `chargePlaced`: 배치 후 실제 셀 수를 잔량 한도 내
  cap 차감하고 **실제 차감량** 반환(음수 불가).
두 경로 모두 차감량 == 요청(또는 잔량 한도) 셀 수로 스펙 일치. 실제 "배치 셀 수" 공급원(래스터라이즈
결과 카운트)은 GameState/main 배선 = sim↔gameplay 경계면이므로 2차 QA에서 통합 검증.
테스트 커버: ink_budget_test "차감 정확성" 3종, ink_controller_test "배치 게이트·차감" 4종.
