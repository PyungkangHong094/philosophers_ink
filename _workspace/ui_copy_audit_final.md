# 현자의 잉크 — UI 마이크로카피 자연스러움 감사 결과

## 어미 톤 결론

**판정: 현재의 "-어라" 해라체 명령형은 기능 문구에 부적합하다. 대안 톤으로 전환 권고.**

근거:
1. **GDD 자체 원칙 위배** — GDD는 "버튼·기능 문구는 평이한 한국어, 연금술 픽션 톤은 타이틀·장식 문구에만"으로 규정한다. 온보딩 목표/조작 배너는 명백한 기능 문구인데 "-어라" 명령형은 픽션 톤(사부의 하대)에 가깝다. 원칙과 충돌한다.
2. **"-어라"는 사부 톤으로 성립하지 못한다** — 한국어에서 "-어라"(해라체 명령)는 사부·현자의 위엄이 아니라 교과서·구령·훈계의 인상을 준다. 위엄 있는 조언 톤은 오히려 "-라"(하라체: 채우라/담으라) 또는 명사형이다. 현재 형태는 어색한 하대로 읽힌다.
3. **텍스트 최소화 원칙과도 상충** — 명령형 어미는 명사형보다 길다. 목표 배너는 명사형(-기)이 가장 짧고, 한국 모바일 퍼즐 관례(캔디크러시·애니팡류 "…개 모으기/채우기")와도 일치한다.

**제안 톤 세트 (일관 적용):**

| 문맥 | 톤 | 예시 |
|------|----|----|
| 목표 배너(온보딩) | **명사형 -기** | 플라스크 ⟨N⟩개 채우기 |
| 조작 가이드 | **명사형 -기** | 선을 그어 길 만들기 |
| 상태·결과 스낵바(설정/실패) | **-어요/-했어요 (하십시오체가 과하면)** | 안내를 초기화했어요 |
| 클리어·실패 타이틀(장식 허용 구간) | **명사형 또는 담백한 서술** | 정제 완료 / 순수가 깨졌어요 |
| 버튼 | 명사형 그대로 | 다음 레벨 / 다시 하기 |

즉, **기능 문구 = 명사형(-기)**, **시스템 알림 = -어요체**, **장식 타이틀만 담백한 서술체 유지**. "-어라" 명령형은 전면 폐기.

부수적으로 **용어 불일치 2건**을 통일 권고: (1) `사운드`(설정) ↔ `소리`(HUD) → **소리**로 통일, (2) `일시정지`(정지 화면) ↔ `멈춤`(HUD 버튼) → **일시정지**로 통일.

## 요약

- **수정 필요 문구: 핵심 15건 + 용어 통일 2건 + 선택 개선 2건.**
- **어미 톤 결론:** "-어라" 해라체 명령형은 사부 톤으로 성립하지 못하고 GDD 기능-문구 원칙과도 충돌 → **기능 문구는 명사형(-기), 시스템 알림은 -어요체, 장식 타이틀만 담백한 서술체**로 3분할 통일.

**상위 5개 수정:**
1. 온보딩 8문구의 "-어라" 명령형 → **명사형(-기)** 전면 전환 (예: `플라스크를 ⟨N⟩만큼 채워라` → `플라스크 ⟨N⟩개 채우기`).
2. `안내를 초기화했다`(스낵바) → **`안내를 초기화했어요`** (해라체 서술 완화).
3. `목표·조작·별점 안내를 처음처럼`(불완전문) → **`목표·조작·별점 안내를 처음부터 다시 보기`** (문장 완결).
4. `잉크를 아낄수록 별이 오른다`(모호) → **`잉크를 아낄수록 별을 더 받는다`** (별점 획득 의미 명료화).
5. 문법·용어 정리: `⟨물질⟩⟨상태⟩로`의 **조사 누락 보완**(`⟨물질⟩을 ⟨상태⟩로`) + **`사운드`/`소리`·`일시정지`/`멈춤` 용어 통일**.
